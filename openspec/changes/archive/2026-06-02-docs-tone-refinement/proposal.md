## Why

The current documentation voice — sardonic, GlaDOS-inspired, absurdist — overshoots the mark. Personality in docs is welcome, but not every paragraph needs a zinger, and the current style risks undermining the tool's credibility for new users evaluating it professionally. The tone needs to be dialed back to something warmer and more professional that still has character.

## What Changes

- Revise all user-facing documentation in `docs/` to use a refined tone: professional and clear, with occasional dry wit, but no longer relentlessly sardonic or absurdist.
- Update the voice requirement in the `user-documentation` spec to reflect the new standard.

## Capabilities

### New Capabilities

_(none)_

### Modified Capabilities

- `user-documentation`: The voice requirement changes from sardonic/GlaDOS/absurdist to professional-with-personality — clear, direct prose where humor is incidental rather than a pervasive stylistic obligation.

## Impact

- All six user-facing doc files: `docs/index.md`, `docs/about.md`, `docs/getting-started.md`, `docs/configuration.md`, `docs/profiles.md`, `docs/focus-vs-devcontainers.md`
- `openspec/specs/user-documentation/spec.md` — the voice requirement and scenarios will be updated
- No code changes; documentation only
